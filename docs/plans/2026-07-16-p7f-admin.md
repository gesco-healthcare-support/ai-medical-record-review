## Goal

Port the Flask admin page (`mrr_ai/templates/admin.html` + `static/admin.js`) to Next: manage the
document categories and their summary prompts. is_admin gated. Plus a net-new reprocess UI
(Adrian's decision - the backend route exists but the Flask page has no reprocess UI).

## Layout (from admin.html)

- Top bar + `.hd-column`: `<h1>Categories & prompts</h1>` + lead + "Add category" button.
- `.hd-table`: ID . Name . Auto-assign (Yes/No) . Active (badge) . Summary prompt
  (Custom/General badge) . actions (`.hd-admin-actions`: Edit / Prompt / Deactivate|Activate).
- Add/Edit category dialog: ID (number, DISABLED when editing - ids immutable) + Name; Description;
  Example titles (textarea, one per line); two `.ev-check` checkboxes (Auto-assign, Active).
- Prompt dialog (`.ev-dialog-wide` + `.ev-mono` 16-row textarea): edit the per-category summary
  prompt; subtitle notes custom vs inheriting the general prompt.
- NEW: "Reprocess a record" card - record-id input + button -> POST reprocess/{id}.

## Backend (already present - admin.py, 7 routes)

- GET `/api/admin/categories` -> [{id,name,description,examples,active,auto_assign,has_summary_prompt}]
- POST `/api/admin/categories` {id,name,description,examples,active,auto_assign} -> 201 (400 dup/bad id)
- PATCH `/api/admin/categories/{id}` (partial; id immutable; active=false is soft-delete)
- GET `/api/admin/prompts/{id}` -> {category_id,text,effective_text,custom}
- PUT `/api/admin/prompts/{id}` {text} (empty -> 400; creates/bumps a custom prompt)
- POST `/api/admin/reprocess/{document_id}` -> {ok} (404 unknown, 400 no reviewed rows, 409 job running)
All superuser-gated (403 non-admin, 401 anon).
