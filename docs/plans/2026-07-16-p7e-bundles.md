# P7e - Category bundles (/diagnostics, /depositions)

status: in-progress
branch: feat/nextjs-fastapi-rewrite
approach: code (UI; verify-after)

## Goal

Port the Flask bundle workspace (`bundle.html` + `bundle-page.js`, routes in
`mrr_ai/blueprints/pages.py`) to Next. One screen, two routes:

- `/diagnostics` - "Diagnostic & Operative", slug `diagnostic-operative`, categories `["3","8"]`
- `/depositions` - "Depositions", slug `depositions`, categories `["9"]`

Pick or upload a record, review it with the SAME identify/review editor as `/records/[id]`,
then download or summarize just the documents whose category is in this page's set.

## Decision (Adrian, modal)

Extract the boot/poll/autosave lifecycle out of `ReviewPageClient` into a shared
`useReviewWorkflow` hook that BOTH the review screen and the bundle page consume. One source of
truth for the identify/poll/autosave logic. Re-verify the review screen after the refactor.

## Layout (from bundle.html + evaluators-ds.css:644-673)

Single `<main>` that stack-scrolls (DS keys off `body[data-bundle-slug]`; set that attr on mount):

1. Picker (`.hd-column` + `.hd-header`): `<h1>{label} builder</h1>` + `.bundle-lead`, a single
   "Upload PDF" control (`.bundle-upload`), then the My-documents table used as a picker
   (open a row -> mount the editor in place; no delete).
2. Shared editor (hidden until a doc is chosen); its own "Summarize" button hidden here.
3. Action card (`.bundle-actions-inner` 760px + `.hd-card.bundle-card`), shown once rows exist:
   title "{label} records", count hint, fields (Patient/DOB, Evaluation type, Attorney firm),
   buttons Download combined PDF (outline) / Summarize to Word (primary) / Choose another (ghost).
   The two download buttons disable when 0 rows match the category set.

## Backend (already present)

- `POST /api/documents/{id}/bundle/pdf`  `{categories, label}` -> streams `{slug}.pdf`
- `POST /api/documents/{id}/bundle/summarize` `{categories, label, patientName, patientdob,
  QMEorAME, lawfirm}` -> streams `{slug}.docx` (sync; 409 if matched > BUNDLE_SUMMARIZE_CAP,
  400 empty categories, 409 no matches)

## Files

New:
- `frontend/hooks/use-review-workflow.ts` - headless lifecycle hook. Signature
  `useReviewWorkflow(documentId: string | null, { enableSummaries?: boolean })`. null id = idle
  (bundle before a pick); enableSummaries=false (bundle) routes done -> editor, never summaries.
- `frontend/lib/bundle-api.ts` - `downloadBundlePdf` / `downloadBundleSummary` (direct fetch ->
  blob, Content-Disposition filename, 401 -> /login).
- `frontend/components/bundle/bundle-page-client.tsx` - picker + embedded editor + action card;
  sets `body[data-bundle-slug]` on mount.
- `frontend/app/diagnostics/page.tsx`, `frontend/app/depositions/page.tsx` - AppBar + client.

Modified:
- `frontend/components/review/review-page-client.tsx` - consume the hook (thin).
- `frontend/components/review/review-editor.tsx` - add `showSummarize?: boolean` (default true);
  make `onSummarize` optional. Bundle hides the built-in Summarize.
- `frontend/components/documents/documents-table.tsx` - make `onIdentify`/`onDelete` optional;
  hide those kebab items when absent (picker mode keeps only Open).
