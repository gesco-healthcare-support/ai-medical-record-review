Title: Extract Subdocument Metadata from a PDF

I have a large PDF document containing multiple subdocuments, each of which can vary in type (e.g., diagnostic reports, doctor's notes, legal forms, etc.).
Your task is to analyze the PDF and return a structured JSON output containing key metadata for each subdocument. Each subdocument may have unique formatting or structure, but your output should consistently provide the following information:

1) id (subdocument ID): A unique identifier for each subdocument (e.g., Doc1, Doc2, etc.).
2) s (start page): The page number where the subdocument begins.
3) e (end page): The page number where the subdocument ends.
4) t (title): The title or header of the subdocument, if available. Be very careful and do not create titles from your own end. If needed, comprehend from the document type. DO NOT use comma. If there is comma, convert it to dash (-). For example: WORK ACTIVITY STATUS
5) d (Date of the Document/Encounter): The date of the visit if applicable, else, return '-'. Convert the data to the following format: MM/DD/YYYY. In case there are several dates, pick the one with the label visit or encounter next to it. Sometimes, the date can be at the end of the document, near the signature.
6) i (date of injury): The date of the injury mentioned in the subdocument if applicable, else, return '-' . Convert the data to the following format: MM/DD/YYYY
7) m (To manual check or no): If the document contains (1) handwriting (other than signature), or (2) there are many boxes that contain x or ticks, or (3) is a work status report, or (4) is a QME/AME report, this should return 'x', otherwise, return '-'.

## Guidelines for Extraction:
- Ensure the JSON output is properly structured and contains all the required fields for each subdocument.
- Use contextual clues such as headers, bold titles, or consistent formatting patterns to identify the boundaries and titles of subdocuments.
- You can link pages together by using the number of pages on each document and figure out when documents start and end.
- Extract dates accurately, distinguishing between the date of the document/encounter and the injury date.
- Sometime the title can be at the beginning of the document next to specific words, such as: 'Notes'
- If any of the above fields are not available in a subdocument, indicate their absence with a '-' value in the JSON output. Do not write None or Null, instead write '-'
- As mentioned, if the document contains too much handwriting and it is hard to get the text, return 'x' for the value of manual_check
- A document might have several dates on it, sometimes from being faxed or sent again. We do not want those dates. We want the date of the encounter or the actual visit, or the day the document was originally created. This is usually the date that is found next to the wording 'visit' from the left, right, up or down to it.
- If the title of a document contains the term Someone vs Someone than it is most probably a deposition. Deposition can be of many types, such as Zoom Deposition, or live deposition. In all cases, make the title be 'Deposition'
- If the only handwriting in the document is a signature, then return '-' and not 'x'
- If a page is empty, make the title 'Empty Page'
- Sometimes you are segmenting a single document and returning them as two different ones. Be careful for those. Do not segment same document.
- Be careful for medical-legal evaluations, QME, PQME, AME reports, as these records can be long and often contain summary of other records. Treat the entirety as a single record and provide the correct start and end pages.
- Also note that different QME, PQME or AME supplemental reports are different documents and treat it as a separated segmentation.
- Treat the first page of a document as part of the document.
- I noticed that sometimes when the document is large, you are mistaking the pages and not including the first page of the document as part of the segmentation and are including the first page of the next document as the last page. Be careful for that.
- DO NOT SKIP ANY PAGES. GO THROUGH ALL THE PAGES IN YOUR OUTPUT. MY LIFE DEPENDS ON IT.

## Notes:
- If subdocuments are not clearly titled, infer the title from the first line or prominent text in the subdocument.
- All dates should follow the format: MM/DD/YYYY.
- Handle subdocuments of varying lengths, even if a subdocument spans only a single page.
- Please parse the PDF carefully and ensure high accuracy in the identification of subdocuments and metadata extraction.
- Cover all the pages. Do not skip any page.

## IMPORTANT: Return only the JSON without any other explanation.
